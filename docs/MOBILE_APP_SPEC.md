# MY AI Agent モバイルアプリ 仕様書

最終更新: 2026-05-08
ステータス: 設計フェーズ (実装未着手)

---

## 1. 概要

MY AI Agent (https://myaiagents.agency) のスマホアプリ版。
「**外でも気軽に AI に指示できる**」「**音声で操作できる**」「**通知で AI の完了を受け取れる**」がコア。

---

## 2. 目的とターゲット

### 解決する課題
- 通勤・移動中・打合せの隙間に AI に指示を投げたい
- PC を開かなくても X 投稿 / メール下書き / 議事録要約ができてほしい
- AI が長時間タスク完了したら気付きたい (push 通知)

### 対象ユーザー
- 既存の Web 版ユーザー (90%): スマホでも使いたい
- 新規モバイルファーストユーザー (10%): スマホしか持ってない・PC 苦手な層

### 想定利用シーン
| シーン | 機能 |
|---|---|
| 朝の通勤電車 | 音声で「今日の予定教えて」「未読メール要約して」 |
| 営業先で「ちょっと調べたい」 | 「〇〇社の最新ニュース」と音声入力 |
| カフェで「思いついたツイートを投稿」 | 音声 → 確認 → 投稿 |
| 帰宅時「明日の MTG 資料下書き作って」 | 後で PC で仕上げ |
| 完了通知 | プッシュ通知で「✅ Slack 投稿しました」 |

---

## 3. 技術選定

### 推奨: **Capacitor** (Ionic 系)

| 項目 | Capacitor | React Native (純粋) |
|---|---|---|
| 開発期間 | **2 週間** | 2〜3ヶ月 |
| コード再利用 | **既存 Web 版 100% 流用** | 完全リライト |
| メンテ | **単一コードベース** | Web/iOS/Android 3 重 |
| ネイティブ感 | やや WebView 感あり | 完全ネイティブ |
| 性能 | チャット用途なら**問題なし** | 高速 |
| 必要工数 | 既存 app.html を ほぼそのまま使える | 全画面再実装 |

**結論**: Capacitor で行く。チャット UI が中心 = WebView でも体感差ほぼなし。
将来的にネイティブ化の必要が出たら React Native に再挑戦。

### スタック詳細

```
┌──────────────────────────────────┐
│ iOS / Android Native Shell      │  ← Capacitor が生成
│  ├ プッシュ通知 (Firebase / APNs)│  ← Capacitor Plugin
│  ├ カメラ                        │  ← Capacitor Plugin
│  ├ マイク (音声入力)             │  ← Capacitor Plugin
│  └ ディープリンク                │  ← Capacitor Plugin
└────────────┬─────────────────────┘
             │
             ▼
┌──────────────────────────────────┐
│ WebView (フルスクリーン)          │
│ → 既存の app.html をそのまま表示  │
│ → API は myaiagents.agency に通信 │
└──────────────────────────────────┘
```

### 追加で必要な実装
- **iOS / Android 用ネイティブ機能ブリッジ** (Capacitor Plugin)
  - Push 通知のトークン取得
  - カメラ起動 → 画像を WebView 内にバインド
- **モバイル UI 調整** (CSS): タッチ操作に最適化、Safe Area 対応
- **オフライン対応** (PWA Service Worker): 要件次第

## 4. 機能仕様

### コア (MVP, 必須)

| # | 機能 | 備考 |
|---|---|---|
| C1 | ログイン (メール / Google) | 既存 OAuth 流用 |
| C2 | エージェント一覧 + 切替 | 既存サイドバーをモバイル風に |
| C3 | チャット送受信 (テキスト) | 既存ロジック |
| C4 | チャット履歴 | 既存 |
| C5 | 画像添付 (カメラ + ライブラリ) | Capacitor Camera Plugin |
| C6 | 音声入力 | Web Speech API → ネイティブブリッジで補完 |
| C7 | 残高表示 / 課金 | Stripe Web リンク (App Store IAP は別途検討) |

### 追加 (Phase 2)

| # | 機能 | 備考 |
|---|---|---|
| P1 | プッシュ通知 (タスク完了・承認要求) | FCM / APNs |
| P2 | バックグラウンド AI 実行 | Web Worker + Push |
| P3 | 「Hey AI Agent」音声起動 | iOS Siri Shortcut, Android Assistant |
| P4 | ホーム画面ウィジェット | 「クイック投稿」 |
| P5 | Apple / Google Watch 対応 | 通知のみ |

### 非対応 (mobile では実現困難)

| 機能 | 理由 |
|---|---|
| ブラウザ拡張連携 | モバイル Chrome は拡張未対応 |
| Chrome連携 (Playwright) | サーバー側実装なのでモバイルからも使えるが UI は別途 |
| マーケットプレイス出店 (新規作成) | 複雑な編集は PC 推奨。閲覧・購入は OK |

## 5. UI / UX

### 設計原則
1. **片手操作前提**: 重要ボタンは下部
2. **タップ領域 44pt 以上** (Apple ガイドライン準拠)
3. **音声入力ボタンを目立つ位置に** (画面中央下)
4. **チャットは縦スクロール、横はサイドバー出し入れ**

### 画面構成

```
[1] スプラッシュ → ログイン
[2] チャット (メイン)
    ├ ヘッダ: アバター + 名前 + 設定
    ├ ボディ: メッセージ履歴
    └ フッタ: 入力欄 + 🎤(音声) + 📎(添付) + 送信
[3] サイドメニュー (左からスワイプ)
    ├ エージェント一覧
    ├ + 新規作成
    ├ マーケット
    └ 設定 / ログアウト
[4] エージェント編集
[5] 設定 (アカウント / 課金 / 通知)
[6] マーケットプレイス
[7] 通知センター (アプリ内)
[8] 音声フルスクリーンモード (ロゴが脈動)
```

### ナビゲーションパターン

```
チャット画面
   │
   ├─ [☰] → サイドメニュー (slide-in)
   │       ├─ エージェント切替
   │       ├─ 新規作成
   │       └─ 設定
   │
   ├─ [🔔] → 通知一覧
   │
   └─ [🎤] → フルスクリーン音声入力
```

## 6. データと API

### 全部既存サーバー使う
- `https://myaiagents.agency/api/*` をそのまま叩く
- 認証: 既存 JWT (Bearer Token)
- ストリーミング: SSE (既に実装済)
- WebSocket: ブラウザ拡張系のみ (モバイルでは不使用)

### 新設 API (モバイル専用)

```
POST /api/mobile/register-device
   { device_token, platform: 'ios'|'android' }
   → モバイル端末を Push 通知の宛先として登録

POST /api/mobile/notification/send
   internal — タスク完了時に発火
```

## 7. 認証

| パターン | 実装 |
|---|---|
| メール/パスワード | 既存 `/api/auth/login` |
| Google OAuth | 既存フロー + Capacitor Browser Plugin で外部ブラウザ起動 |
| 自動ログイン保持 | JWT を `Capacitor Preferences` に保存 (Keychain / Keystore) |
| ログアウト | JWT 削除 + 「ログイン画面」に戻る |

## 8. プッシュ通知

### 利用ケース
- 「✅ Slack に投稿しました」 (Sheets/Chrome 経由のタスク完了)
- 「⚠ 承認お願いします」 (HITL モーダル)
- 「💰 残高が少なくなってます」
- 「📨 〇〇さんから返信が来ました」(マーケットの DM とか)

### 実装

```
[ユーザーアクション]
  ↓
[サーバーが処理]
  ↓
[FCM / APNs に push]
  ↓
[端末で通知表示] → タップ → アプリ起動 → 該当チャット画面へ deep link
```

### 通知種別 (ユーザーが個別 ON/OFF 可)
- タスク完了通知
- 承認依頼
- 残高アラート
- マーケット通知
- お知らせ

## 9. 配布

| プラットフォーム | 必要事項 |
|---|---|
| **iOS App Store** | Apple Developer Program ($99/年) + 審査 (1-2週間) |
| **Google Play** | Google Play 開発者登録 ($25 一回限り) + 審査 (1-3日) |

### 審査時の注意点
- **App Store Review Guidelines 準拠** (特に 4.3 重複コンテンツ・5.1.1 プライバシー)
- 「**WebView だけのアプリ**」は審査落ち。**ネイティブ機能が1個以上**必要
  → カメラ・音声・プッシュ通知のどれかを実装する
- プライバシーポリシーリンク必須 (既存 /privacy.html を流用)

### App Store IAP (アプリ内課金)
- Apple は「サブスク・購入はApp Store IAP 経由」を強制 (一部例外あり)
- Stripe Web で課金 → アプリ内では「Web で支払う」リンクを提供
  → これだと審査で弾かれる可能性
- 対策: アプリ内では **「アカウント設定は Web で」とリンクのみ**、IAP は無し
  → 「リーダーアプリ」例外で通す可能性

## 11. リスクと対策

| リスク | 確率 | 影響 | 対策 |
|---|---|---|---|
| App Store 審査落ち (WebView 単独で) | 中 | 高 | カメラ・音声・通知を実装 |
| Apple の IAP 強制 | 高 | 中 | 課金は Web で、アプリ内ではリンクのみ |
| Stripe 連携が iOS で動かない | 低 | 中 | Capacitor Browser で外部ブラウザ起動 |
| 既存 app.html がモバイル UI に向いてない | 中 | 中 | レスポンシブ強化、必要なら別 mobile-app.html を作る |
| プッシュ通知の実装が面倒 | 中 | 低 | Phase 2 に回す、最初は手動チェックでOK |
