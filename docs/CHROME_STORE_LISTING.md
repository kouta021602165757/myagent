## 単一目的の説明 (Single Purpose)

```
本拡張は MY AI Agent サービスにペアリングされたユーザーのリクエストを受け、
ユーザーが指示した特定の Web サイト操作 (URL を開く・要素クリック・テキスト入力等)
を実行するためだけに存在します。

This extension exists solely to execute web operations (opening URLs,
clicking elements, typing text) on behalf of a user who has paired it
with their MY AI Agent service account, only when the user explicitly
requests these actions through the MY AI Agent chat interface.
```

## ホスト権限の説明 (Permission Justification)

各権限について「なぜ必要か」をストアフォームに記入する必要があります。下記を貼り付け:

### `<all_urls>` (host_permissions)
```
ユーザーが操作対象として指定したあらゆる Web サイト (X / Slack / Gmail /
社内 SaaS 等) でコマンドを実行する必要があるため。バックグラウンドで
ページを監視することはなく、ユーザーが MY AI Agent チャットで指示した
瞬間のみページの読み取りと操作を行います。
```

### `tabs`
```
AI からの「指定 URL を新しいタブで開く」「アクティブタブを切り替える」
「現在開いているタブの一覧を取得する」コマンドを実行するため。
```

### `scripting`
```
content.js を対象ページに動的注入し、AI 指示によるクリック・入力・
読み取り操作を実行するため。
```

### `storage`
```
ペアリングトークン (32 バイトのランダム値) を chrome.storage.local に
保存するため。これがないと毎回ペアリングし直す必要が生じます。
```

### `notifications`
```
将来的にユーザー承認が必要な操作の OS 通知表示用 (今は未使用)。
```

### `activeTab`
```
ユーザーが拡張アイコンをクリックした時にアクティブタブの状態を確認するため。
```

## 審査通過のコツ

1. **権限を明確に説明**: 上記の Permission Justification を漏れなく書く
2. **プライバシーポリシーへのリンクを忘れない**: 必須項目
3. **スクリーンショットは「使ってる場面」を見せる**: 起動画面だけだと審査落ちしやすい
4. **不可逆操作の確認モーダルがあることをスクリーンショットに含める**: Google は「ユーザーの安全」を最重要視
